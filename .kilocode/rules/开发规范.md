# 开发规范.md

Vue3+TS 技术栈的官方最佳实践，项目中的命名规范，涵盖目录、文件、变量、函数、类等所有核心场景。

## 指导原则

结合你指定的命名规则和Vue3+TS技术栈的官方最佳实践，为你整理了一套完整且可落地的命名规范，涵盖目录、文件、变量、函数、类等所有核心场景。

## 一、核心命名规则总览

|命名对象|命名风格|示例|
|---|---|---|
|目录名称|中划线（kebab-case）|`src/views/user-center`|
|文件名称|中划线（kebab-case）|`user-profile.tsx`、`api-request.ts`|
|变量/常量|小驼峰（camelCase）|`userInfo`、`apiBaseUrl`|
|函数/方法|小驼峰（camelCase）|`getUserList()`、`formatDate()`|
|类/接口|大驼峰（PascalCase）|`UserService`、`ApiResponse`|
|函数参数|小驼峰（camelCase）|`function fetchData(pageNum: number)`|
|函数返回值|小驼峰（camelCase）|`const userData = fetchUser()`|
|类成员变量|小驼峰（camelCase）|`class User { userName: string }`|
|Vue组件（TSX）|大驼峰（PascalCase）|`UserProfile.tsx`（文件名中划线，组件名大驼峰）|
|Pinia状态/动作|小驼峰（camelCase）|`state: { userId: string }`、`actions: { updateUser() }`|
|路由命名|中划线（kebab-case）|`path: '/user-detail'`、`name: 'userDetail'`|
## 二、分场景详细规范

### 1. 目录命名规范

- 强制使用**中划线（kebab-case）**，全部小写

- 目录层级清晰，遵循Vue官方推荐的项目结构

- 禁止使用下划线、大写字母、空格或特殊字符

### 2. 文件命名规范

- 所有文件名称使用**中划线（kebab-case）**，全部小写

- Vue组件（TSX）文件：`组件名-描述.tsx`（如`user-profile.tsx`）

- 工具函数文件：`功能描述.ts`（如`date-format.ts`）

- 类型定义文件：`模块名-type.ts`（如`user-type.ts`）

- Pinia仓库文件：`模块名-store.ts`（如`user-store.ts`）

- 禁止使用中文、大写字母、下划线，文件名简洁且表意明确

**错误示例**：

- `UserProfile.tsx`（使用大写）

- `user_profile.ts`（使用下划线）

- `用户信息.tsx`（使用中文）

**正确示例**：

- `user-profile.tsx`

- `user-info.ts`

- `date-format.ts`

### 3. 变量/常量命名规范

- 普通变量：**小驼峰（camelCase）**，表意明确，避免缩写（除非是通用缩写如`id`、`url`）

- 常量：全部大写，下划线分隔（补充规范，AI项目常用），如`API_BASE_URL`

- 布尔类型变量：前缀使用`is`/`has`/`can`，如`isLogin`、`hasPermission`

- 数组变量：后缀使用`List`/`Array`，如`userList`、`roleArray`

- 对象变量：使用语义化名称，如`userInfo`、`apiConfig`

**示例**：

```TypeScript

// 普通变量
const userName = '张三';
const userAge = 25;
const apiBaseUrl = 'https://api.example.com';

// 常量
const MAX_RETRY_COUNT = 3;
const TOKEN_EXPIRE_TIME = 3600;

// 布尔变量
const isLogin = false;
const hasPermission = true;
const canEdit = false;

// 数组/对象
const userList = [{ id: 1, name: '张三' }];
const userInfo = { id: 1, name: '张三', age: 25 };
```

### 4. 函数/方法命名规范

- 函数/方法名称：**小驼峰（camelCase）**

- 动词开头，表意明确，如`get`/`set`/`fetch`/`update`/`delete`/`format`

- 异步函数：前缀可加`async`（可选），如`asyncGetUserInfo`

- 事件处理函数：前缀`handle`，如`handleButtonClick`、`handleFormSubmit`

- 钩子函数：前缀`use`（Vue3官方规范），如`useRequest`、`useAuth`

**示例**：

```TypeScript

// 普通函数
function getUserInfo(userId: number) {
  return api.get(`/user/${userId}`);
}

// 异步函数
async function asyncFetchUserList(pageNum: number) {
  const res = await api.get('/users', { params: { pageNum } });
  return res.data;
}

// 事件处理函数
function handleLoginSubmit(formData: LoginForm) {
  // 处理登录逻辑
}

// 自定义钩子（Vue3官方规范）
function useRequest<T>(url: string) {
  const data = ref<T | null>(null);
  const loading = ref(false);
  
  const fetchData = async () => {
    loading.value = true;
    try {
      const res = await axios.get(url);
      data.value = res.data;
    } catch (err) {
      console.error('请求失败:', err);
    } finally {
      loading.value = false;
    }
  };

  return { data, loading, fetchData };
}
```

### 5. 类/接口命名规范

- 类/接口名称：**大驼峰（PascalCase）**

- 类名：名词开头，如`UserService`、`ApiClient`

- 接口名：前缀`I`（可选，TS常用规范），如`IUser`、`IApiResponse`

- 类成员变量：小驼峰，私有成员前缀加`_`（补充规范）

**示例**：

```TypeScript

// 接口定义
interface IUser {
  userId: number;
  userName: string;
  userAge: number;
}

interface IApiResponse<T> {
  code: number;
  message: string;
  data: T;
}

// 类定义
class UserService {
  // 私有成员变量
  private _apiBaseUrl: string;
  
  constructor(baseUrl: string) {
    this._apiBaseUrl = baseUrl;
  }
  
  // 类方法（小驼峰）
  async getUserInfo(userId: number): Promise<IUser> {
    const res = await axios.get(`${this._apiBaseUrl}/user/${userId}`);
    return res.data;
  }
}
```

### 6. Vue3 TSX组件规范（官方推荐）

- 组件文件名：中划线（`user-profile.tsx`）

- 组件名称：大驼峰（`UserProfile`），与文件名语义对应

- Props定义：使用`defineProps`，类型注解优先，Props名称小驼峰

- 事件定义：使用`defineEmits`，事件名中划线（Vue官方规范）

- 组件内部变量/方法：小驼峰

- 禁止使用`setup`语法糖外的选项式API，遵循组合式API规范

**示例组件（user-profile.tsx）**：

```TypeScript

import { defineProps, defineEmits, ref } from 'vue';

// Props定义（小驼峰）
const props = defineProps<{
  userId: number;
  userInfo: {
    userName: string;
    userAge: number;
  };
  isEdit: boolean;
}>();

// 事件定义（事件名中划线，Vue官方规范）
const emit = defineEmits<{
  (e: 'update-user', userData: { userName: string; userAge: number }): void;
  (e: 'cancel-edit'): void;
}>();

// 内部变量（小驼峰）
const editForm = ref({
  userName: props.userInfo.userName,
  userAge: props.userInfo.userAge,
});

// 内部方法（小驼峰）
const handleSave = () => {
  emit('update-user', {
    userName: editForm.value.userName,
    userAge: editForm.value.userAge,
  });
};

const handleCancel = () => {
  emit('cancel-edit');
};

// 组件导出（大驼峰）
export default function UserProfile() {
  return (
    <div class="user-profile">
      <el-form model={editForm.value} disabled={!props.isEdit}>
        <el-form-item label="用户名">
          <el-input v-model={editForm.value.userName} />
        </el-form-item>
        <el-form-item label="年龄">
          <el-input-number v-model={editForm.value.userAge} min={1} />
        </el-form-item>
        {props.isEdit && (
          <el-form-item>
            <el-button onClick={handleSave}>保存</el-button>
            <el-button onClick={handleCancel} type="text">取消</el-button>
          </el-form-item>
        )}
      </el-form>
    </div>
  );
}
```

### 7. Pinia状态管理规范

- 仓库文件：中划线（`user-store.ts`）

- 状态变量：小驼峰（`userId`、`userName`）

- Action方法：小驼峰（`updateUserInfo`、`fetchUserList`）

- Getter计算属性：小驼峰（`userFullName`、`isAdmin`）

**示例（user-store.ts）**：

```TypeScript

import { defineStore } from 'pinia';
import { fetchUserInfo, updateUserInfo } from '@/api/user/user-info';

// 仓库名称：小驼峰
export const useUserStore = defineStore('userStore', {
  state: () => ({
    // 状态变量：小驼峰
    userId: 0,
    userName: '',
    userAvatar: '',
    isLogin: false,
    userRoleList: [] as string[],
  }),
  getters: {
    // Getter：小驼峰
    isAdmin(): boolean {
      return this.userRoleList.includes('admin');
    },
    userFullName(): string {
      return `${this.userName}(${this.userId})`;
    },
  },
  actions: {
    // Action：小驼峰
    async fetchUserInfoAction(userId: number) {
      try {
        const res = await fetchUserInfo(userId);
        this.userId = res.data.userId;
        this.userName = res.data.userName;
        this.userAvatar = res.data.userAvatar;
        this.isLogin = true;
      } catch (err) {
        console.error('获取用户信息失败:', err);
      }
    },
    async updateUserInfoAction(userData: { userName: string; userAvatar: string }) {
      await updateUserInfo(this.userId, userData);
      this.userName = userData.userName;
      this.userAvatar = userData.userAvatar;
    },
    logoutAction() {
      this.userId = 0;
      this.userName = '';
      this.isLogin = false;
    },
  },
});
```

### 8. 路由配置规范

- 路由路径：中划线（`/user-center`、`/user-detail`）

- 路由名称：小驼峰（`userCenter`、`userDetail`）

- 组件导入：对应TSX文件（中划线）

- 路由参数：小驼峰（`userId`、`pageNum`）

**示例（router/index.ts）**：

```TypeScript

import { createRouter, createWebHistory } from 'vue-router';

// 懒加载组件
const Dashboard = () => import('@/views/dashboard/dashboard.tsx');
const UserCenter = () => import('@/views/user-center/user-center.tsx');
const UserProfile = () => import('@/views/user-center/user-profile.tsx');
const UserSetting = () => import('@/views/user-center/user-setting.tsx');

const routes = [
  {
    path: '/',
    redirect: '/dashboard',
  },
  {
    path: '/dashboard', // 路径：中划线
    name: 'dashboard',  // 名称：小驼峰
    component: Dashboard,
  },
  {
    path: '/user-center',
    name: 'userCenter',
    component: UserCenter,
    children: [
      {
        path: 'profile/:userId', // 参数：小驼峰
        name: 'userProfile',
        component: UserProfile,
        props: true,
      },
      {
        path: 'setting',
        name: 'userSetting',
        component: UserSetting,
      },
    ],
  },
];

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes,
});

export default router;
```

## 三、补充注意事项

1. **命名一致性**：同一语义的名称在整个项目中保持一致（如用户信息统一用`userInfo`，不混用`userData`）

2. **避免冗余**：命名简洁，避免无意义的前缀/后缀（如`userUserInfo`）

3. **TS类型规范**：类型别名/接口使用大驼峰，如`type UserInfo = { userName: string }`

4. **注释规范**：关键函数/类/组件必须添加注释，说明功能、参数、返回值

5. **Vue官方规范补充**：

    - 组件属性优先使用`kebab-case`（模板中），如`<user-profile user-id="1" />`

    - 事件名统一使用`kebab-case`，如`@update-user="handleUpdate"`

    - 指令缩写优先使用（`v-bind`→`:`，`v-on`→`@`）

### 总结

1. **基础命名规则**：目录/文件用中划线（kebab-case），变量/函数/参数/类成员用小驼峰（camelCase），类/接口用大驼峰（PascalCase）；

2. **Vue3+TS适配**：TSX组件文件名用中划线，组件名用大驼峰，Props/事件遵循Vue官方规范（事件名中划线）；

3. **生态适配**：Pinia仓库/路由命名结合核心规则，状态/Action用小驼峰，路由路径用中划线、名称用小驼峰。
